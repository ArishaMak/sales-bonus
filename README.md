ShopAnalytics — Веб-приложение аналитики продаж
Курсовой проект по дисциплине Базы данных и системы управления базами данных
#################################################################

Описание проекта
Информационная система ShopAnalytics предназначена для автоматизации учета продаж в интернет-магазине, управления товарным ассортиментом и анализа эффективности работы сотрудников. Проект реализует полный цикл обработки данных: от хранения в реляционной базе данных до визуализации ключевых показателей эффективности (KPI) и расчета премиальных выплат.

#################################################################

Основные функции
Учет транзакций: Регистрация и централизованное хранение данных о покупках, товарах, клиентах и персонале.

Аналитический дашборд: Мониторинг динамики выручки и прибыли в реальном времени с использованием графических инструментов.

Расчет KPI и бонусов: Алгоритмическое вычисление процента выполнения плана и автоматическое начисление премий.

Управление данными: Многокритериальный поиск, фильтрация и серверная пагинация для оптимизации работы с большими датасетами.

Безопасность: Многоуровневая система аутентификации с применением хэширования bcrypt и настройка политик Content Security Policy (CSP).

#################################################################

Технологический стек
Frontend: JavaScript (ES Modules), HTML5, CSS3 (модульная архитектура), Chart.js, библиотека XLSX.

Backend: Node.js, фреймворк Express.js.

База данных: MySQL 8.0 (реляционная модель, транзакционные механизмы, триггеры аудита).

#################################################################

Структура проекта
1. Серверная часть (Backend)
backend/server.js — Точка входа приложения, конфигурация Express и middleware.

backend/db.js — Модуль управления пулом соединений с СУБД MySQL.

backend/routes/ — Определение маршрутов API для авторизации и получения аналитических данных.

backend/services/kpiService.js — Ядро бизнес-логики, выполняющее расчет агрегированной статистики на стороне сервера.

2. Клиентская часть (Frontend)
frontend/index.html — Базовая разметка интерфейса и контейнеры модальных окон.

frontend/src/main.js — Центральный контроллер, обеспечивающий инициализацию и связь модулей.

frontend/src/modules/api.js — Интерфейс взаимодействия с серверным API (fetch-запросы).

frontend/src/modules/charts.js — Компонент визуализации данных на базе Chart.js.

frontend/src/modules/auth.js — Модуль управления сессиями и правами доступа.

frontend/css/ — Стилистическое оформление (переменные, макеты сетки, темы оформления).

#################################################################

Ключевые модули и алгоритмы
Бизнес-логика (Backend Service)
Реализована посредством асинхронных процедур взаимодействия с базой данных:

calculateSellerStats: выполняет агрегацию данных из таблиц транзакций, вычисляет чистую прибыль и коэффициент выполнения плана.

getPeriodKPI: формирует структурированные наборы данных для построения временных рядов.

Слой визуализации (Charts Module)
renderCharts(stats): инициализирует графические объекты (линейные и кольцевые диаграммы), обеспечивая корректное отображение статистики.

Система управления памятью: включает механизмы деструктуризации старых экземпляров графиков для предотвращения утечек ресурсов.

Аналитический модуль (Frontend Logic)
analyzeSalesData: преобразует необработанные данные (raw data) от сервера в прикладные структуры для табличного отображения.

exportToExcel: реализует экспорт сформированных отчетов в формат .xlsx для дальнейшей обработки.

Инструкция по запуску
1. Подготовка базы данных
Создайте базу данных и импортируйте схему:

SQL

CREATE DATABASE sales_bonus;
mysql -u root -p sales_bonus < data_dump.sql
2. Конфигурация системы
Создайте в корневом каталоге файл .env и настройте параметры подключения:

Фрагмент кода

DB_HOST=localhost
DB_USER=имя_пользователя
DB_PASSWORD=пароль
DB_NAME=sales_bonus
PORT=5000
3. Запуск сервера
Bash

npm install
npm run dev
Интерфейс будет доступен по адресу: http://localhost:5000

Ключевые требования и архитектура
Безопасность: Применение заголовков CSP для предотвращения XSS-инъекций.

Архитектура: Соблюдение принципа разделения ответственности (Separation of Concerns) между уровнем представления, бизнес-логикой и слоем данных.

Интерактивность: Реализация асинхронного обмена данными (AJAX) для динамического обновления интерфейса.
